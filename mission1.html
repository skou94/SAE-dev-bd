Contexte général

Kdou est une entreprise qui vend des produits d'épicerie fine du monde entier, exclusivement en France. Bien qu'elle réalise un bon chiffre d'affaires, elle cherche à se démarquer de ses concurrents et à améliorer sa rentabilité. L'entreprise souhaite également adopter une image plus « éco-friendly » pour attirer des clients. Notre mission est donc d'analyser ses données et de proposer des solutions pour l'aider à atteindre ses objectifs.

Pour cela, nous avons reçu un fichier Excel contenant la base de données. Cette base de données contient plusieurs tables.

Analyse des relations entre les tables

Les questions 1 et 2 de cette mission ont pour but d'analyser les relations entre les tables. Comme nous étions sur Excel, les tables n'avaient pas de relations définies, il a donc fallu analyser quelle table pouvait correspondre à une clé primaire ou à une clé étrangère.

Exemple de Clé Primaire

Par exemple, dans la table fournisseur, on peut prendre comme clé primaire le N° fournisseur, car c'est un numéro unique pour chaque fournisseur et que sa valeur ne peut être nulle.

Exemple de Clé Étrangère

Pour les clés étrangères, on pourrait par exemple utiliser le code catégorie dans la table produit, ce qui fait le lien entre la table produit et la table Catégorie.

Analyse des Clés Primaires et Étrangères

Les questions 1 et 2 ont permis de comprendre les relations entre les différentes tables, en particulier en identifiant les clés primaires et étrangères.

Dans la table fournisseur, la clé primaire est le N° fournisseur, un identifiant unique pour chaque fournisseur. Dans la table produit, la clé étrangère est le code Catégorie, qui fait le lien avec la table Catégorie.
